# Negotiation and Prioritization for Engineers

## Negotiation Is Not “Winning”
Professional negotiation aligns **scope**, **time**, **risk**, and **quality** with stakeholders who have different incentives.

## Useful Frameworks
- **MoSCoW**: Must / Should / Could / Won’t for backlog triage.
- **RICE** or **WSJF**: compare features when capacity is constrained.
- **Fixed time, fixed scope trap**: if both are fixed, **quality or honesty** suffers—surface the triangle early.

## Saying No Constructively
Template:
- “We **can** ship X by Friday if we **defer** Y and accept **risk** Z (data migration not load-tested). Which trade-off do you prefer?”

## Evidence-Based Pushback
Bring **metrics**, **spikes**, or **prototypes** instead of pure opinion—especially for performance or security concerns.

## Urgent vs Important
- **Incident**: drop planned work with explicit communication.
- **Stakeholder loudness** ≠ priority—use agreed **product goals** as tie-breaker.
